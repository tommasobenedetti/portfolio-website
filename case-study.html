MyFlix






Concept


MyFlix is a simple movie app, where users can search for movies, and get further information about directors and the movie genres. The movie data are collected from an API, users can login/register and can edit their own user data!

Project's Duration?


At the beginning of this project, I struggled with React as it is a quite complex framework. This project took me around 4 months to finish.





Project's Tools


Backend MongoDB, NodeJS (Mongoose, Express, CORS, Passport, bodyParser, Bcrypt, Morgan), Heroku, Postman GitHub
Frontend React, Redux, Javascript, Parcel, Axios GitHub




Credits

This Web Application wouldn't be possible without the guidance and help of my mentor Mohamad Muaaz Nashawi and my tutor Julio José López Aldazoro.





Overview

This Web Application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information and create a list of their favorite movies.





Features


Return a list of all the movies to the user. Return addictional data (description, genre, director, image URL) about a single movie by title. Return data about a genre (description) by name/title. Return data about a director.





Server-Side


This project was built starting from the Server-Side by mean of a RESTful API, which would pass the information to the front-side of the website.

Simplicity Breakdown: An API is a software that sends information back and forward between a website/app and a user.

A REST API uses HTTP requests to receive, send, create, and delete data or files. In other words, via a REST API, a client is able to make changes to or access data from a database they otherwise wuoldn't have access to.







Client-Side


It is the visual area of the website where the backend requests are displayed. I've used React together with Redux to build it. The user can access further info about movies,directors and genres. They can also register, login, and change user information.





Backend Goals


  • Install NodeJS with npm, and start building the project.
  • Use Express to create API points, in order to get the movies data lon the frontend.
  • CRUD implementation (Create, Read, Update, Delete), with Postman.
  • Implementing a CORS policy. CORS allows us to recognize where the request are coming, and if they are coming from trustable sources.
  • Password hashing, and Authentication with Passport Middleware.

Frontend/Client Goals


  • Choosing between Class or Functional components in React.
  • Create low and high-level components using React's lifecycle methods.
  • Distinguish between Props and States.
  • Implement methods of authentication and form validation between views.
  • Redux implementation, Redux stores interaction logic for users.

Biggest Challenges?


  • Initially the user-edit and registration areas were not working. The user would save it's personal info, the page would refresh, and no info would be saved.
  • Some debugging was required using Google Chrome DevTools and Visual Studio Code. My tutor and my mentor also helped me a lot to fix this issue.
  • For some time a CORS error message kept appearing on the Chrome DevTools when opening the website. This was caused by a function error in my code that required a deep search to be discovered.
  • I've experienced some terminal related errors at the begining on the backend side of the project. These errors were due to package updates, the errors were occuring mainly with Parcel.

Further Improvements


  • The website will go through some design changes, as I believe in continuous improvement.
  • For instance, additional movies will be added in the future.

Conclusion


MyFlix was the first website I've ever built. As it combines the backend with the client-side, it turned out to be a very challenging at the beginning. Nevertheless, as I got deeper into the project, I got more and more comfortable with it.

Apart from learning React, with its quirks and benefits, one of my favorite parts was using Postman. This is an incredible tool that allows you to test all the requests as if you were a user registering on the website, to be sure that all of them are functional. It worked as a perfect bridge between the backend and frontend.


The excitement is real, I cannot wait to add more movies and directors to my website!
